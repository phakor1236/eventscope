# ex1 — L2 語義級:純記憶體事件庫(MemoryEventStore)

**練習核心**:`src/db.ts` 的冪等寫入,但**沒有 Postgres 幫你**。原版的防雙記帳
是資料庫送的(`UNIQUE` 鍵 + `ON CONFLICT DO NOTHING`);這題你要用一個 `Map`
親手實作出一模一樣的行為——證明「冪等」是你的設計,不是資料庫的贈品。

## 與原版的差異

| | 原版(src/db.ts) | 這題 |
|---|---|---|
| 儲存 | Postgres 表 | 記憶體裡的 Map |
| 唯一鍵 | `UNIQUE (tx_hash, log_index)` 宣告給 DB | **你自己設計鍵、自己查、自己擋** |
| 重複時 | DB 靜默跳過 | 你的碼靜默跳過 |

## 行為規則(測試都會打)

1. `insert(rows)`:收一批事件,存進庫,**回傳「這次新增了幾筆」**。
2. **冪等(靈魂)**:同一個 `(tx_hash, log_index)` 不管來幾次、跨幾批,
   庫裡永遠只有一筆,而且**只有第一次算進回傳值**。
3. **同批內的重複也要擋**:一批裡塞兩筆相同鍵的,只存一筆、只數一次。
4. **first-write-wins**:重複鍵再來時,就算內容(args)不同,**保留最早那筆,
   新的丟掉**——這是 `DO NOTHING` 的語義,不是「更新」(那是 UPSERT,L4 才登場)。
5. 鍵是**兩個零件合起來**的:同 `tx_hash` 不同 `log_index` 是兩筆不同事件,
   同 `log_index` 不同 `tx_hash` 也是。只用其中一個零件當鍵,測試會抓到。
6. `has(txHash, logIndex)`:問「這張身分證收過沒」。
7. 空批 → 回傳 0,庫不動。

## 必須維持的不變量

1. 任何操作序列跑完:**庫裡的筆數 = 出現過的不同 (tx_hash, log_index) 數**。
2. 所有 `insert` 回傳值加總 = 庫裡的筆數(每筆事件恰好被「算」一次)。
3. 已存的資料永不被後來者改寫(first-write-wins)。
